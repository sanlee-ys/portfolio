SYS-004 · SYS-006

Decouple with contracts, not shared code

The repos talk over frozen HTTP contracts with explicit versioning, and both sides carry contract tests in CI. The system behaves as one thing while each piece stays independently deployable.

Tradeoff: more upfront ceremony, so a renamed field fails a build instead of breaking production in silence.

Why it mattered

Decoupling without a contract is just hope. The hard part lives in the seams between the pieces, not the pieces themselves. Full writeup →

EVAL · v1 → v2

I rebuilt the eval until it could tell me I was wrong

v1 graded the model on 300 snippets it generated itself, which measures consistency, not correctness. v2 replaced that with 54 hand-labeled real snippets, cross-checked by an Opus judge.

The result: domain accuracy fell from 97.3% to 88.9% on real text, and that lower number is the trustworthy one.

Read the numbers

Category accuracy actually rose (79.0% to 88.9%) because real vocabulary separates more cleanly. Domain fell because the synthetic set was trivially easy to grade. The gold set is 54 snippets, hand-labeled. Full eval →

EVAL · BM25 rejected

I added retrieval grounding, measured it, and cut it

A negative result is still a result. I added BM25 lexical retrieval to ground each call, expecting a clear win.

The result: +1.9% category accuracy, +0.0% on domain, and a flip analysis showed it fixed about as many calls as it broke. It did not earn the jump to embeddings, so it did not ship.

The discipline

Measure first, escalate only when the eval says it pays. Here it said do not bother. Full eval →

EVAL · PROMPT reverted

A prompt that read better and measured worse, reverted

A change that sounds right can move the decision boundary the wrong way, and intuition cannot see that. I sharpened the procurement-versus-industry rule so it read cleaner to a human.

The result: category accuracy dropped from 79.0% to 76.7% and industry recall from 0.217 to 0.100. I reverted it.

Why measurement won

The sharper rule gave the model a cleaner excuse to dump borderline company stories into the wrong bucket. Only the eval could see that. Full eval →

SYS-002

Gate model choice on evidence, not vibes

Sonnet by default across the system. Escalate to Opus only where an eval shows the quality gain pays for itself.

Tradeoff: you have to run the eval to justify the tier instead of reaching for the biggest model.

What the eval showed

The classifier's ceiling was label ambiguity, not model horsepower, so paying roughly 1.7× for a larger model would have bought nothing. Full writeup →

ADR-002

Force structured output with tool-use, not prompt parsing

The model is called with a forced tool schema (both fields required, strict enums), so an invalid label is rejected at the API layer before it reaches application code.

Tradeoff: a few extra tokens and a more verbose call, in exchange for app code with no defensive parsing.

How it held up

The schema becomes the contract. Out-of-enum responses were rare (once in 300) and a single re-sample handled them. Full writeup →

ARCH right-sized

I built the loop on Kafka, then cut it

The honest move was to right-size it, not defend it. I built the enrichment loop on Kafka to learn event-driven architecture properly: an idempotent consumer, at-least-once delivery, offset management.

Then I cut it. A single-user workload with derived, idempotent tags never needed a distributed log, so it is now an in-process FastAPI BackgroundTask.

What that trades away

You give up durability across a crash and replay, neither of which this workload needs yet. The day it does, the move is an outbox and a queue, not a reflex reach for a broker. Full writeup →

About, honestly
