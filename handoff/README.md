# handoff/

Working drafts staged for pickup by a session with broader repo access than
whatever session wrote them.

This portfolio repo is often the only one in scope for a given Claude Code
session (see the GitHub-access scoping note in `CLAUDE.md`/session config).
When work belongs in a *different* repo in the system (`kb-agent`,
`notes-api`, `defense-news-classifier`, `learning-notes`) but gets drafted
here anyway, it's parked in this folder rather than an ephemeral scratchpad,
so it survives to be picked up later — by a local session, or a future
remote session with that repo in scope.

**Not part of the public site.** Nothing here is linked from the site's nav;
it's a staging area, not a page. `ROADMAP.md` links to anything parked here
that's still open, so it doesn't go orphaned the way `learning/` briefly did.

## Picking one up

1. Read the file's own header — each one states what it is, what it's
   unverified against, and where it's meant to land.
2. Verify its claims against the real source in the destination repo before
   treating it as final (the whole point of parking it, not just trusting it).
3. Once it's committed to the destination repo, delete it from here and drop
   its row from the table above — this folder holds drafts in flight, not an
   archive.
