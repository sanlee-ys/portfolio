Convention

ADR / SYS-NNN

A short written record of one architecture decision: context, choice, tradeoff, alternatives considered. Numbered so it can be cited instead of re-argued.

In this system: ADR-NNN are decisions scoped to one repo; SYS-NNN are decisions that cross repo boundaries — the contracts wiring notes-api, the classifier, and kb-agent together.

AI

Agent

Software that chains tool calls toward a goal instead of answering in one shot: pick a tool, read the result, decide the next step, repeat until done or stuck.

In this system: kb-agent searches the knowledge base and synthesizes a grounded answer, rather than guessing from what the model already knows.

Systems

BackgroundTask

A way to run a follow-up step right after a request without standing up a message queue. The caller gets its response immediately; the extra work runs in the same process a moment later.

In this system: notes-api uses FastAPI's BackgroundTask to call the classifier and write its labels back as tags — the loop was built on Kafka first, then cut when the workload didn't earn a broker. Full decision →

Retrieval

BM25

A keyword-scoring algorithm for search: it ranks documents by how often and how distinctively they contain the words in your query, no model required. Fast, cheap, a solid baseline.

In this system: BM25 was tried as retrieval grounding for the classifier, measured, and cut — it moved category accuracy +1.9% but domain +0.0%, and fixed about as many calls as it broke. Full decision →

Classification

Sorting text into a fixed set of labels using a model, instead of hand-written rules.

In this system: the classifier assigns two labels to each defense-news item — a category (procurement, operations, policy, technology, industry) and an operational domain (air, land, sea, cyber, space, multi). Full writeup →

Systems

Contract

An explicit, versioned interface between two systems: what one side promises to produce, what the other is allowed to assume. Neither side needs to know how the other works internally, only that the shape holds.

In this system: three frozen contracts (SYS-004, SYS-005, SYS-006) wire the repos together, each with contract tests in CI, so a renamed field fails a build instead of breaking production. Full decision →

AI

Eval / eval-driven development

A scored test set with known-correct answers, used to check whether a change actually helped — instead of trusting that it reads better.

In this system: this is the whole discipline the site is built to demonstrate. Every prompt change, retrieval experiment, and model choice here was measured against an eval, including the ones that failed and got reverted. Full decision →

AI

F1 / precision / recall

The numbers used to score a classifier instead of trusting a skim. Precision: of everything labeled X, how much was actually X. Recall: of everything that was actually X, how much got caught. F1: the balance of the two — useful exactly when one alone would flatter the model.

In this system: see the classifier evidence table for the real numbers, synthetic vs. hand-labeled. Full table →

Systems

Idempotency

An operation that produces the same result whether it runs once or five times — safe to retry, no side effects pile up.

In this system: it's why the classify-and-tag writeback didn't need Kafka's delivery guarantees. The tags it writes are derived and idempotent, so re-running the step just re-arrives at the same state. Full decision →

AI

RAG (retrieval-augmented generation)

Instead of asking a model to answer from memory, retrieve the relevant documents first, then hand only those to the model as grounding for its answer. Fixes the model's biggest weakness — it doesn't know your data — without retraining it.

In this system: kb-agent is the RAG layer, retrieving from the knowledge base before it answers a query. Full system →

AI

Tool use / structured output

Forcing a model to respond through a defined schema — a function call with typed, required fields — instead of parsing free text and hoping the format holds.

In this system: the classifier's tool schema makes an invalid label a rejected API call, not a runtime bug. Out-of-enum responses were rare (once in 300). Full decision →
