Colophon

How this site was built, because a site about being honest should be honest about how it was made.

This site was built the way I would run any small project with an agent: I set the direction, the contracts, and the bar; an AI (Claude) did most of the typing; and I verified the output against the real repos before it shipped. The decisions and the tradeoffs are mine. The keystrokes mostly are not, and pretending otherwise would undercut the one thing this site is about.

That claim gets the same standard as everything else here: evidence over assertion. So the honest version is not that I was in control, but the record of it, the times I overruled the agent and the times the evidence overruled me. Knowing where you ended up is not the same as proving how you got there, and the record is the part worth keeping.

Directing a capable agent well, setting the contracts, catching the overreach, refusing the convenient answer, is the work I think matters as the tools get stronger. This site is the practice rep, and this page is me saying so out loud.

The ledger

A sample of that record, mined from the actual session transcripts. The shape is the point: not “go build it, come back when it’s done,” but a steady run of course corrections, mine and the agent’s, across the whole build. The dates and words are real, only tightened. A list that only flattered me would be the bluff.

  1. Jun 19
    The agent tried to burn tokens.

    It wanted to re-run 60 API calls to regenerate results that were already in memory. I stopped it. “Sounds like an excuse to use more tokens.”

  2. Jun 20
    A memory it could not keep.

    It said it would keep a rule in mind; I asked how, since it cannot across sessions. We wrote the rule into the global config instead. “How will you keep it in mind?”

  3. Jun 21
    It undersold itself.

    It told me I would hand-collect about forty articles; I asked whether it could just fetch them. It could, and auto-pulled most of the corpus. “Are you not able to crawl the Internet…?”

  4. Jun 21
    And sometimes its case won.

    It pushed back on my docs format, Markdown over RST; I weighed the reasoning and took it. The judgment cuts both ways. “if you feel it’s overkill, so be it.”

  5. Jun 22
    It almost clobbered a parallel branch.

    It started editing a repo where another session of mine held unmerged work; I pulled it off before it touched anything. “I’ll just have you resume once the learner session’s done.”

  6. Jun 27
    Kafka.

    It insisted the system still used Kafka, off stale local clones. It did not. That rewrote the copy and became a standing rule to verify before asserting. “it’s not even using Kafka anymore. No?”

  7. Jun 28
    “Nothing’s broken” was not true.

    It told me the site was fine on mobile; it felt cramped to me, and digging found a real bug crushing the gutters on every deep page.
