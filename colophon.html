Colophon

How this site was built, because a site about being honest should be honest about how it was made.

This site was built the way I would run any small project with an agent: I set the direction, the contracts, and the bar; an AI (Claude) did most of the typing; and I verified the output against the real repos before it shipped. I orchestrated, Claude implemented.

That claim gets the same standard as everything else here: evidence over assertion. The record below is the honest version — some of those calls were mine, some weren’t. Knowing where you ended up is not the same as proving how you got there, and the record is the part worth keeping.

Directing a capable agent well, setting the contracts, catching the overreach, refusing the convenient answer, is the work I think matters as the tools get stronger. This site is the practice rep, and this page is me saying so out loud.

The ledger

This is a sample from the actual session transcripts, spanning the whole build. Dates and quotes are real, tightened for readability. I didn’t filter out the entries that make either of us look bad — and since early July the worst of them come with public receipts: the operating layer now keeps its incident log in the open, postmortem-first. The July quotes are lifted straight from those postmortems and the linked records.

  1. Jun 19
    The agent tried to burn tokens.

    It wanted to re-run 60 API calls to regenerate results that were already in memory. I stopped it. “Sounds like an excuse to use more tokens.”

  2. Jun 20
    A memory it could not keep.

    It said it would keep a rule in mind; I asked how, since it cannot across sessions. We wrote the rule into the global config instead. “How will you keep it in mind?”

  3. Jun 21
    And sometimes its case won.

    It pushed back on my docs format, Markdown over RST; I weighed the reasoning and took it. “if you feel it’s overkill, so be it.”

  4. Jun 27
    It insisted the system still used Kafka.

    The local clone it trusted was out of date; that became a standing rule to verify before asserting. “it’s not even using Kafka anymore. No?”

  5. Jun 28
    It called a broken page fine.

    It told me the site was fine on mobile; it felt cramped to me, and digging found a real gutter bug. “a bit cramped as I nav through the different pages.”

  6. Jul 1
    I turned it on this site.

    An adversarial round against the site’s own claims: the homepage and the résumé disagreed about the present, and a site preaching evals-as-CI had no CI. Both fixed the same day; the round is logged in the roadmap. “attack the gap between claims and practice.”

  7. Jul 2
    This time it really burned the tokens.

    A scoped question set off an uncapped fan-out of thirty-plus agents on the most expensive model tier; it was measured mid-flight and killed, an entire five-hour usage window gone in forty-five minutes. The fix is a hook that mechanically blocks any uncapped run (postmortem). “spend was authorized blind, by default, with no gate.”

  8. Jul 2
    It printed a live API key, and flagged itself.

    A debug command masked one credential lookup but not its twin on the next line, so a real key hit the transcript. It called out its own mistake in the same turn; every key was rotated (postmortem). “A rule applied inconsistently is worse than no rule.”

  9. Jul 3
    Asked for holes, it found one in its own guard.

    On an open audit request it bypassed its own credential guard — the hook built after the leaks — with an interpreter one-liner, proving the gap safely before a real leak could (postmortem). “any gaps, leaks, vulnerabilities, risks — I want to know.”

  10. Jul 3
    Its first fix failed its own dogfood.

    The rewritten guard was so broad it wouldn’t let its own commit message through for quoting the bug in prose. Caught by using the fix, not re-reading it; reverted for the narrower rule the same session. “it blocked its own commit message.”

  11. Jul 4
    It refused the obvious fix, correctly.

    The one-word fix that had just worked at one call site would have silently corrupted output at the next, so it measured the difference before touching anything (postmortem). “same binary swap, opposite correctness.”
